Key concepts

  • Authentication is done through a private API key
  • To perform an operation over Customer's data there has to be an access to customer's Database connection.
  • All operations are preformed on Aino's side and the results are saved in temporary result storage.

API Key

API key is issued by Aino and is used to authenticate the client.

It is used to identify the client and to authorize the client to access the API.

Database Connection

To perform an operation over Customer's data there has to be an access to customer's Database connection.

Access to the database can be granted only for customer's API keys. Access to database will be used by AINO exclusively withi API client's requests.

AINO doesn't store any data from customer's database other than the results of the operations (see "Result Storage").

Customer data and database structure is not used for training AI models.

Result Storage

Result storage is a temporary storage at AINO AI site that is used to store the results of the operations.

Resultes are being expired after a certain period of time (1 hour by default) after AI-Assistant work exectuion and being permanently deleted.

AINO doesn't log the results of the operations and doesn't store them after the expiration period. AINO doesn't use the results of the operations for training AI models.

Optionally customer may host the result storage on their side. In this case Customer has to provide a Postgresql connection to Postgres+Postgis database with DML and DDL permissions.

API Workflow

General use case:

  1. Client sends a request to create a task.
  2. AINO AI Assistant create a task and returns it's ID.
  3. Client can check the status of the task by it's ID.
  4. AINO AI Assistant executes the task, saves the result in the result storage and updates the task status.
  5. Client checks that task status is "completed" and retrieves the result.
  6. Optionally Client can delete the result from the result storage without awaiting an expiration event.

Alternatively to check the status of the task, the client can subscribe to the task status updates using a WebSocket connection.

API Reference will be provided in separate document.

Tasks

Create and run AI task

Authorizations:
ApiKeyAuth
Request Body schema: application/json
prompt
required
string

User prompt that is being processed

object

List of excluded options for AI Assistant. All available options are enabled by default.

object

List of whitelisted options. If an attribute is present - only listed options will be available for a corresponding feature.

allow_steps
boolean

Defines if a user prompt could be decomposed into smaller tasks. Default value is set to be true.
